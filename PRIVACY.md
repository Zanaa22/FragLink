# FragLink Privacy Policy

Effective July 18, 2026 · Policy version 1

FragLink has no user account, analytics, advertising, or remote OCR service.

## Purpose and limited use

FragLink handles only the temporary page content and user-created fragrance data
needed to search for fragrances, recognize visible fragrance text, use tools on a
supported fragrance page, and maintain the user’s local Scent Shelf. FragLink’s
use of information received from Chrome APIs complies with the Chrome Web Store
User Data Policy, including its Limited Use requirements.

FragLink does not sell user data, use it for advertising or creditworthiness,
make it available for humans to read, or transfer it to third parties. The only
external disclosure is the user-directed search described below, where Chrome
opens the destination selected by the user with the confirmed fragrance query.

## Local OCR

FragLink Scan runs only after the user clicks it and draws a rectangle on the
visible active browser tab. The screenshot is cropped and processed by bundled
English and Spanish OCR code on the user’s device. Screenshot and crop pixels
are not uploaded, written to extension storage, or retained after recognition.

## Local storage

FragLink uses Chrome local extension storage for provider preferences, result-tab
behavior, optional recent-search text, and user-created Scent Shelf items. A
small session-storage entry may remember a result tab ID when the user chooses
the reused-tab behavior. Recent searches can be disabled and cleared. Shelf
items can include the exact supported fragrance-page URL when the user saves
from page tools or the page-level context menu. Shelf items can be edited,
deleted, exported, and restored by the user.

## External searches

A fragrance name leaves FragLink only when the user chooses a search or price
destination. FragLink constructs that destination’s URL and asks Chrome to open
it. The destination site then receives the query under its own privacy terms.
FragLink sends no background telemetry.

## Invoked page tools and clipboard

Page tools and the supported-page Save context-menu action read a small title
snapshot only after the user invokes them. The snapshot is not retained. When
the user explicitly saves the fragrance, FragLink keeps the reviewed name and
exact supported page URL in the local Scent Shelf. Clipboard writing occurs
only when the user clicks Copy and no persistent clipboard permission is
requested.

## User controls

The Settings page controls recent-search storage. The Scent Shelf page supports
per-item deletion, complete replacement through validated JSON import, and JSON
or CSV export.

Privacy questions can be sent to the verified support email shown on FragLink’s
Chrome Web Store listing.
